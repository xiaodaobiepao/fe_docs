# [源码篇] 手写Promise等API源码

不仅仅是前端，程序员的进阶不能只停留在增删改查，或者各种API的调用中，必须得深入到源码级别，理解源码不仅对工作中使用对应的api有帮助，并且学习优秀的源码，也能提升自己的编码水平。下面介绍几种api源码实现，仅仅大体原理，可能达不到实际生产需要，轻喷……

# Promise

Promise 是一种异步编程解决方案，为了解决传统回调产生的回调地狱而生，ES6将其写入了语言标准。Promise 对象有以下特点：

  • 对象的状态不受外界影响。Promise 对象代表一个异步操作，有三种状态：pending，fulfilled，rejected，只有异步操作的结果可以改变状态，不会受其他状态影响。
  • 对象状态一旦改变，就不会再变，只可能从pending变为fulfilled，或者从pending变为rejected，状态不可逆。
  • 可以支持链式操作，并且在上一个then状态未变时，不会执行后面的链式回调。
  • 回调返回值为Promise时，要后续的then链接到返回的Promise上

根据以上规范，我们先完成只有resolve版本的

function Promise(fn) {
  this.state = 'pending'
  this.data = ''
  this.callbacks = []

  const resolve = (value) => {
    console.log(this)
    const fn = () => {
      if (value && (typeof value === 'object' || typeof value === 'function')) {
        if (value.then && typeof value.then === 'function') {
          // 如果判断是否是Promise实例，可以用 value instanceof Promise
          // 这里是兼容thenable情况，即实现的then函数的对象

          // 将当前resolve传入，以便当value是Promise对象时，可以延续后续then到value上
          value.then(resolve)
          return
        }
      }
      this.state = 'fulfilled'
      this.data = value
      while (this.callbacks.length) {
        const callback = this.callbacks.shift()
        this._handle(callback)
      }
    }
    setTimeout(fn, 0) // 防止用户没有异步操作，直接同步调用resolve
  }

  fn(resolve)
}

Promise.prototype.then = function (fulfilledFn) {
  return new Promise((resolve) => {
    this._handle({ // 将执行then产生的新的Promise中的resolve放入当前Promise实例中
      fulfilledFn,
      resolve
    })
  })
}

Promise.prototype._handle = function (cbObj) {
  if (this.state === 'pending') {
    // 此时当前实例状态还未改变，先加入callbacks中，待resolve或者reject函数改变状态并调用
    this.callbacks.push(cbObj)
    return
  }

  if (this.state === 'fulfilled') {
    // 状态为成功
    const ret = cbObj.fulfilledFn(this.data) // 调用回调
    cbObj.resolve(ret) // 把返回值传给下一个then
  }
}

上述实现了简易的resolve版的Promise，关键在于then函数中，then函数返回一个新的Promise实例，并且将传入的fulfilledFn回调和resolve回调存储在上一个环节的Promise实例中，在上一个Promise实例执行resolve并状态变为fulfilled时，才会执行then里面的回调，并调用resolve使下一个Promise实例状态变为fulfilled。

如果fulfilled回调返回的是Promise，则进行特殊处理，直到返回的Promise状态改变后才进入下一链。

接下来加上reject

function Promise(fn) {
  this.state = 'pending'
  this.data = ''
  this.callbacks = []

  const resolve = (value) => {
    const fn = () => {
      if (value && (typeof value === 'object' || typeof value === 'function')) {
        if (value.then && typeof value.then === 'function') {
          // 如果判断是否是Promise实例，可以用 value instanceof Promise
          // 这里是兼容thenable情况，即实现的then函数的对象

          // 将当前resolve传入，以便当value是Promise对象时，可以延续后续then到value上
          value.then(resolve)
          return
        }
      }
      this.state = 'fulfilled'
      this.data = value
      while (this.callbacks.length) {
        const callback = this.callbacks.shift()
        this._handle(callback)
      }
    }
    setTimeout(fn, 0) // 防止用户没有异步操作，直接同步调用resolve
  }

  const reject = (err) => {
    const fn = () => {
      // reject就不用支持传入Promise了
      this.state = 'rejected'
      this.data = err
      while (this.callbacks.length) {
        const callback = this.callbacks.shift()
        this._handle(callback)
      }
    }
    setTimeout(fn, 0)
  }

  fn(resolve, reject)
}

Promise.prototype.then = function (fulfilledFn, rejectedFn) {
  return new Promise((resolve, reject) => {
    this._handle({ // 将执行then产生的新的Promise中的resolve放入当前Promise实例中
      fulfilledFn,
      rejectedFn,
      resolve,
      reject
    })
  })
}

Promise.prototype._handle = function (cbObj) {
  if (this.state === 'pending') {
    // 此时当前实例状态还未改变，先加入callbacks中，待resolve或者reject函数改变状态并调用
    this.callbacks.push(cbObj)
    return
  }

  const callback = this.state === 'fulfilled' ? cbObj.fulfilledFn : cbObj.rejectedFn
  if (!callback) {
    this.state === 'fulfilled' ? cbObj.resolve(this.data) : cbObj.reject(this.data)
    return
  }
  let ret
  try {
    ret = callback(this.data)
  } catch(err) {
    cbObj.reject(err)
    return
  }
  cbObj.resolve(ret)
}

# finally

还有finally，我们希望无论是成功还是失败，都要有一个最终都会执行的回调

Promise.prototype.finally = function (done) {
  return this.then(done, done)
}

以上，Promise的基本实现已经完成了，虽然有一些不完善的地方，比如实际上Promise是微任务，优先级要比宏任务要高，这里就不细究了。

Promise还提供了一些常用的API，例如Promise.all可以批量并发执行异步任务，并且都resolve后再触发Promise.all的fulfilled状态。还有Promise.resolve，Promise.reject等API

# Promise.resolve

Promise.resolve会返回一个Promise实例，该实例状态为fulfilled，传入的参数不同，Promise.resolve还会做不同的处理。具体可以参考阮一峰的ES6标准入门。

Promise.resolve = function(value) {
  if (!value) {
    return new Promise(resolve => resolve)
  }
  if (value instanceof Promise) {
    return value
  }
  if ((typeof value === 'object' || typeof value === 'function') && typeof value.then === 'function') {
    return new Promise(resolve => {
      value.then(resolve)
    })
  }
  return new Promise(resolve => resolve(value))
}

# Promise.reject

Promise.reject也会返回一个Promise实例，该实例状态为rejected，与Promise不同的是，Promise.reject会将传入的值原封不动的作为reject的理由，并变成后续方法的参数。

Promise.reject = function(err) {
  return new Promise((resolve, reject) => reject(err))
}

# Promise.all

Promise.all = function(fn) {
  let index = 0
  let result = []
  return new Promise((resolve, reject) => {
    if (fn.length === 0) {
      return resolve(result)
    } else {
      for (let i=0;i < fn.length;++i) {
        fn[i].then(data => {
          result.push(data)
          index++
          if (index === fn.length) {
            resolve(result)
            return
          }
        })
        .catch(err => {
          reject(err)
          return
        })
      }
    }
  })
}

# Function.prototype.call

call可以强绑定函数执行时this的指向。

这是别人的一个方法

Function.prototype.call = function (context, ...args) {
  var context = context || window;
  context.fn = this;

  var result = eval('context.fn(...args)');

  delete context.fn
  return result;
}

但我认为它有个问题，如果传入的context本来就有个fn属性，会导致，被覆盖掉。所以可以修改下。

Function.prototype.call = function(context, ...args) {
  context = context || window
  const obj = Object.create(context)
  obj.fn = this
  const res = obj.fn(...args)
  obj = nulls
  return obj.self(...args)
}

# Function.prototype.apply

同理apply实现:

Function.prototype.apply = function(context, args) {
  context = context || window
  const obj = Object.create(context)
  obj.fn = this
  const res = obj.fn(...args)
  obj = nulls
  return obj.self(...args)
}

# Function.prototype.bind

bind的实现方式要稍微复杂点，因为bind返回的是一个函数，还要考虑new的操作：

Function.prototype.bind = function(context, ...args) {
  context = context || window
  const self = this
  function bindFn(...args2) {
    // 如果是new bindFn 操作，new 操作符产生的obj（即当前this），需要保留，如果不是，才会绑定context为this指向
    return self.apply(this instanceof bindFn ? this : context, [...args, ...args2])
  }
  bindFn.prototype = Object.create(this.prototype)
  return bindFn
}
